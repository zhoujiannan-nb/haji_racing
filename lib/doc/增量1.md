# 新增赛道功能
简介：该功能主要以展示赛道列表,点击进入赛道详情介绍赛道简介, 在点击开始跟跑可以在用户到达赛道定位起点的情况下,根据预制的起点电子围栏和判断规则，开始计时，在到达终点电子围栏的时候，完成计时

# 技术依赖
- Flutter Map 插件（flutter_map、latlong2）配合高德瓦片服务
- 定位权限：需要在 Android/iOS 配置文件中声明

# 赛道列表界面
- 在主界面的车辆的边上,点击赛道图标进入
- 展示赛道的名称, 赛道的长度, 发布的时间
  
## 赛道详情界面
- 展示赛道的长度,名称,简介
- 展示赛道的轨迹缩略图（由flutter_map配合高德瓦片生成）

## 赛道跟跑详情
- 展示赛道的名称
- 展示当前的定位点在地图的位置（3D导航模式，车头朝上）
- 当前实时计时（格式：MM:SS.ms）
- 显示当前位置与起点的距离提示
- 点击开始按钮前：验证用户是否在起点电子围栏范围内（半径50米），不在则提示"请前往起点"
- 点击开始按钮后：进入定位逻辑，开始1秒一次连续定位
- 停止按钮：手动停止计时，本次轨迹不保存

# 数据库设计

## 版本升级
- 当前数据库版本：3
- 新版本：4（添加用户表和赛道相关表）

## 用户表 (users)
```sql
CREATE TABLE users (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  username TEXT NOT NULL DEFAULT 'race',
  createdAt TEXT NOT NULL DEFAULT (datetime('now'))
)
```
- 说明：app启动时自动创建默认用户 'race'，整个app生命周期只有一个用户

## 赛道表 (tracks)
```sql
CREATE TABLE tracks (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,
  description TEXT,
  length REAL NOT NULL,  -- 赛道长度（米）
  startLatitude REAL NOT NULL,  -- 起点纬度
  startLongitude REAL NOT NULL,  -- 起点经度
  startRadius REAL NOT NULL DEFAULT 50.0,  -- 起点电子围栏半径（米）
  endLatitude REAL NOT NULL,  -- 终点纬度
  endLongitude REAL NOT NULL,  -- 终点经度
  endRadius REAL NOT NULL DEFAULT 50.0,  -- 终点电子围栏半径（米）
  thumbnailUrl TEXT,  -- 缩略图URL（高德静态图API）
  publishedAt TEXT NOT NULL DEFAULT (datetime('now')),
  createdAt TEXT NOT NULL DEFAULT (datetime('now'))
)
```
- 说明：赛道数据在app编译时预置，通过SQL脚本初始化

## 轨迹记录表 (track_records)
```sql
CREATE TABLE track_records (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  userId INTEGER NOT NULL,
  trackId INTEGER NOT NULL,
  startTime TEXT NOT NULL,  -- 开始时间
  endTime TEXT,  -- 结束时间
  duration REAL,  -- 总时长（秒）
  status TEXT NOT NULL DEFAULT 'incomplete',  -- incomplete/completed
  FOREIGN KEY (userId) REFERENCES users(id),
  FOREIGN KEY (trackId) REFERENCES tracks(id)
)
```

## 轨迹点表 (track_points)
```sql
CREATE TABLE track_points (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  recordId INTEGER NOT NULL,
  latitude REAL NOT NULL,
  longitude REAL NOT NULL,
  speed REAL,  -- 速度（km/h）
  timestamp TEXT NOT NULL,
  sequence INTEGER NOT NULL,  -- 点位顺序
  FOREIGN KEY (recordId) REFERENCES track_records(id)
)
```
- 说明：本期先实现数据存储，轨迹回放功能下期实现

# 逻辑说明

## 赛道列表
- 赛道数据从sqlite中获取,在app编译的时候就已经写好了
- 只有点击查看详情的功能
  
## 赛道详情界面
- 赛道的基础信息从上述数据库获取
- 赛道的缩略图由flutter_map配合高德瓦片生成（使用轨迹坐标绘制）
  
## 赛道跟跑界面

### 初始化阶段
- 通过flutter_map展示实时定位点和当前地图预览（使用高德瓦片）
- 计算并显示当前位置与起点的距离
- 如果距离起点 > 50米，禁用开始按钮并提示"请前往起点"

### 开始跟跑流程
1. 用户点击开始按钮
2. 验证：检查当前位置是否在起点电子围栏内（距离 <= 50米）
   - 如果不在：弹出提示"请前往起点附近再开始"
   - 如果在：继续下一步
3. 创建轨迹记录（status='incomplete'）
4. 启动连续定位（1秒/次）
5. 开始计时器

### 计时触发逻辑
- 持续监听定位数据
- 触发条件：**同时满足**以下两个条件
  1. 当前位置在起点电子围栏内（距离起点 <= 50米）
  2. 车速 > 15 km/h
- 触发后：正式记录起始时间，开始保存轨迹点

### 运行中逻辑
- 每秒获取一次定位数据
- 保存每个定位点到 track_points 表
- 实时更新计时器显示
- 持续检测是否到达终点

### 结束逻辑
- **自动结束**：当检测到当前位置在终点电子围栏内（距离终点 <= 50米）
  - 停止计时
  - 停止定位
  - 更新轨迹记录：设置 endTime、duration、status='completed'
  - 保存所有轨迹点
  
- **手动停止**：用户点击停止按钮
  - 停止计时
  - 停止定位
  - **删除本次轨迹记录**（不保存任何数据）
  - 返回赛道详情页

### 异常处理
- 所有异常情况（定位失败、权限拒绝、网络异常等）：直接终止操作，提示"操作失败，请重试"
- 不做复杂的降级处理，以简化实现
