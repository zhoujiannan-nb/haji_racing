# 对赛道轨迹的设定升级

当前赛道数据结构只支持起点,终点,和其他,需要进行升级

## 升级设定
赛道轨迹的基础结构要变更为： 起点（多边形电子围栏）,检查点（多边形电子围栏）+ 设定规则（比如落在检查点比如有2个点速度低于40km/h这样的逻辑）, 终点（多边形电子围栏）
赛道其他名称 + 简介就不要变了

## 赛道制作
旧: 旧版本赛道是直接写在app初始化插入里面的
新版本： 我需要你再HAJI_RACING的目录下创建一个目录,此目录结合高德的sdk或者js实现一个web网页,可以再上面进行电子围栏标注, 终点和起点的设定, 检查点围栏的标注, 特殊规则（速度）的设定, 最后导出成json, 丢给app,app首次编译或者启动会把json读取到自己的sqlite中,变成赛道。原来的赛道全部删除

## 额外说明
这版本升级会涉及旧数据,不用管他们,无需做任何兼容迁移逻辑,我自己会清空他们的

## 使用说明

### Web 编辑器使用
1. 打开 `track_editor_web/index.html`
2. 配置高德地图 API Key（需要替换 YOUR_AMAP_KEY）
3. 在浏览器中打开即可使用
4. 绘制完成后下载 JSON 文件

### 导入赛道到 App
1. 将 JSON 文件放入 `assets/tracks/` 目录
2. 更新 `assets/tracks/tracks_index.json` 添加文件名
3. 运行 `flutter pub get`
4. 重新编译运行 App

### 清空旧数据
如需清空所有旧赛道数据，可以在应用中调用：
```dart
final importService = TrackImportService();
await importService.clearAllTracks();
```

## 注意事项

1. **高德地图 API Key**: Web 编辑器需要配置有效的高德地图 API Key 才能正常工作
2. **数据库迁移**: 首次运行会自动执行数据库迁移，旧数据会被清除
3. **JSON 格式**: 必须遵循指定的 JSON 格式，参考 example_track.json
4. **多边形要求**: 每个多边形至少需要 3 个顶点
5. **Assets 配置**: 每次添加新的 JSON 文件后需要更新 tracks_index.json
